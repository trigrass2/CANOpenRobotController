5 Welcome to [CORC Project](https://github.com/UniMelbHumanRoboticsLab/CANOpenRobotController)! Before submitting pull requests, please make sure that you have **read the whole guidelines**. 11 We are very happy that you consider implementing CORC with a currently supported or new platform! This repository is referred to and used by researchers and developers from a wide range of backgrounds around the globe. Being one of our contributors, you agree and confirm that:

23 #### Code

44 #### New Directory guidelines
45 
46 - We recommend adding files to existing directories as much as possible.
47 - Use Upercase words without a separator ( no spaces or `"-"` allowed )

59 #### Commit Guidelines
60 
61 - It is recommended to keep your changes grouped logically within individual commits. Maintainers find it easier to understand changes that are logically spilt across multiple commits. Try to modify just one or two files in the same directory. Pull requests that span multiple directories are tedious to manage and may be rejected.
62 
63 ```
64 git add file_xyz.cpp
65 git commit -m "your message"
66 ```
67 
68 Examples of commit messages with semantic prefixes:
69 
70 ```
71 fix: xyz algorithm bug
72 feat: add xyx algorithm, class xyz
73 test: add test for xyz algorithm
74 docs: add comments and explanation to xyz algorithm
75 ```
76 
77 Common prefixes:
78 
79 - fix: A bug fix
80 - feat: A new feature
81 - docs: Documentation changes
82 - test: Correct existing tests or add new ones

84 #### Documentation
85 
86 - Write self documenting code whenever possible and use DOXYGEN commenting when needed.
87 - Document method and class definitions using DOXYGEN in .h files.
88 - Make sure you put useful comments in your code. Do not comment things that are obvious.
89 - Please avoid creating new directories if at all possible. Try to fit your work into the existing directory structure. If you want to create a new directory, then please check if a similar category has been recently suggested or created by other pull requests.
90 - If you have modified/added documentation, please ensure that your language is concise and contains no grammar errors.
