1 # Creating CPU utilization log from command line (LINUX)
2 This is a guide for a basic CPU utilisation experiment for a CORC Application.
3 
4 It takes output from the default Linux command `top` - sampling the current state of the system at regular intervals, and reduces the resulting output to only the CPU utilisation for the given application. This can then be post-processed to find an average across those intervals.
5 
6 ## Determine PID of EXO_APP_2020
7 `top` requires the Process Identification (PID) of the application under investigation. This can be found with the following command.
8 
9 `pidof EXO_APP_2020`
10 
11 We refer to the result of this as `<PID>`.
12 
13 ## Use of the top command
14 We use the following options for the top command:
15 
16 -p : only display top information for the specified PID process
17 -b : batch mode (this formats the output such that it can be written to file)
18 -d : Delay i.e. intervals between iterations
19 
20 The suggested command format is as follows:
21 
22 `top -p <PID> -b -d 0.1 > top.txt`
23 
24 This produces a file which logs the state of the machine at 1 second intervals. This interval can be changed by changing the `0.1` parameter.
25

57 ## Sample only PID info lines to a file
58 
59 The following command removes all lines apart from the ones which include the PID. Note that it's possible that this also catches some 'incidental' (e.g. if your PID is 1, it will capture all lines with a 1 in it --- this can optimised with a better command).
60 
61 `cat top.txt | grep <PID> > topPIDonly.txt`
62 
63 This produces an output similar to below:
64 
65 ```
66  2304 root 20 0 19572 1120 1016 S 18.8 0.2 0:11.43 EXO_APP_2020
67  2304 root 20 0 19572 1120 1016 S 27.3 0.2 0:11.46 EXO_APP_2020
68  2304 root 20 0 19572 1120 1016 S 20.0 0.2 0:11.48 EXO_APP_2020
69  2304 root 20 0 19572 1120 1016 S 18.2 0.2 0:11.50 EXO_APP_2020
70  2304 root 20 0 19572 1120 1016 S 20.0 0.2 0:11.52 EXO_APP_2020
71  ```

73 ## Display only CPU utilization from each line and pipe to file w/ nl for time period of each data point
74 
75 We can combine this with `cut -c (columns)` command to take only the relevant lines (those associated with CPU usage) - note that the relevant columns (48-52) have been manually specified.
76 
77 `cat top.txt | grep <PID> | cut -c 48-52| nl >> cpuLog.txt`
78 
79 (nl adds number line to each line entry - this is optional, and it may be easier to process if you do not use this line).
80 
81 This produces an output similar to below, which can then be processed to calculate the average CPU utilisation.
82 
83 ```
84 1 18.8
85 2 27.3
86 3 20.0
87 .
88 .
89 .
90 ```
